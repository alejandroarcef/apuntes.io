Git

Iniciando

Debemos configurar el nombre de usuario y el correo. Para esto ejecutamos:

git config --global user.name "nombre" git config --global user.email "correo"

Para configurar el editor y que la consola espere mientras se edite un archivo ejecutamos:

git config --global core.editor "code --wait"

Para configurar una cosa rara ahí ejecutamos:

git config --global core.autocrlf input

Para ver la configuración de git ejecutamos el comando:

git config -l

Para iniciar el repositorio se debe ir hasta la carpeta creada previamente y estando dentro ejecutar el comando: git init

automáticamente se creará la carpeta .git dentro de esta estará toda la información y configuración relacionada con este repositorio.

Agregar cambios

Después de crear cualquier archivo o hacer cualquier cambio git sabrá que algo se adiciono o modifico pero aún no estará agregado al repositorio.

El staging es un “lugar” o “estado" temporal que se guarda en memoria ram, cuando los archivos pasan al estado staging pasan de estar untrack a track.

Para adicionar el archivo nuevo o modificado al staging ejecutamos el comando:

git add [file]

Para adicionar todos los nuevos archivos y todos los cambios al staging ejecutamos:

git add .

Para volver el archivo a el estado untrack ejecutamos:

git reset head [file]

Para sacra un archivo del estado track y que git no lo reconozca se usa el comando:

git rm --cached [file] git checkout -- [file]

Si por alguna razón accidentalmente tienes archivos en la carpeta del repositorio que no quieres incluir y quieres borrarlos para esto ejecutas primero el comando:

git clean --dry-run

Para borrarlos definitivamente ejecutas:

git clean -f --dry-run

Para borrar un archivo de git y tambien del disco duro ejecutamos:

git rm -f [file]

Para guardar momentáneamente los cambios del staging sin hacerle commit y poder cambiar de rama le hacemos un stash para esto ejecutamos el comando:

git stash

Para ponerle un mensaje a esos cambios guardados en stash ejecutamos:

git stash save "mensaje"

Para ver los stash guardados ejecutamos:

git stash list

Puedes crear una nueva rama con estos cambios guardados en stash ejecutamos:

git stash branch [nombre]

Para recuperar los cambios que están en stash ejecutamos:

git stash pop

Commit

Para pasar los archivos de staging a realmente estar en el repositorio se debe hacer un commit esto se hace con el comando:

git commit -m "mensaje del commit"

Para unificar el git add y el git commit ejecutamos el comando:

git commit -am "mensaje del commit"

Automaticamente a cada commit se le asignara un id este es un indicador de commit

Para agregar cambios que están en staging a un commit anterior se usa el comando:

git commit --amend

Cada commit es una nueva versión del proyecto, aunque normalmente para definir una nueva versión se le asigna un tag a un commit especifico ya que no es practico establecer una versión por cada cambio, sino que cuando se hagan cierta cantidad de cambios significativos y que funcionen correctamente entonces se decide establecer una nueva versión.

Para agregar un tag ejecutamos el comando:

git tag -a [nombre del tag] -m "mensaje" [id del commit]

Para ver los tags existentes ejecutamos:

git show-ref --tags

Para subir los tags a github ejecutamos:

git push origin :refs/tags/ndeltag

Para borrar un tag ejecutamos:

git tag -d [nombre del tag]

Ver los commit

Para ver el estado de los archivos ejecutamos el comando:

git status

Para comparar los cambios que no estén en staged ejecutamos:

git diff

Para ver lo que hay en una rama que no hay en otra ejecutamos:

git diff [branchB] [branchA]

Para comparar los cambios que esten en staging ejecutamos:

git diff --staged

Para hacer una comparación entre commits y ver los cambios ejecutamos el comando:

git diff [id del commit] [id del commit]

Para ver solo los commit de un archivo especifico se ejecuta:

git log [file]

Para ver todos los commit ejecutamos:

git log

Para ver los commit de una rama que no estan en la otra ejecutamos:

git log [branchB] [branchA]

Para ver los commit mas resumidos ejecutamos:

git log --oneline

Para ver un commit especifico se ejecuta:

git log -s [id del commit]

Para ver los commit por autores ejecutamos:

git shortlog

Para ver los commit con un grafico visual ejecutamos:

git log --graph

Para ver los commit con mas detalles ejecutamos:

git log --stat

Para ver la historia completa (TODO) ejecutamos el comando:

git reflog

Para saber quien escribió un archivo ejecutamos

git blame [file]

Para saber quien escribió cada línea del archivo ejecutamos:

git blame [file] -L5,13

Para buscar cosas dentro del repo usamos el comando:

git grep "palabra"

Para resetear todo a un commit anterior ejecutamos el comando:

git reset [id del commit] --hard

Para configurar alias ejecutamos:

git config --global alias.[palabra] codigo

Ramas

Por defecto al crear el repositorio se crea la rama master, esta es la rama principal, se debería crear otra rama que normalmente se llama development y en esta empezar a desarrollar el proyecto y cuando te asegures de que los códigos son correctos y funcionen iras actualizando la rama master.

A la rama master, que es la principal en lo posible no se le pasaran errores estos deben de ser solucionados en la rama development, de esta también pueden derivar diversas ramas en dependencia del proyecto, normalmente para los pequeños bug que se pueden solucionar rápidamente se crea una rama normalmente llamada hotfix.

Para crear una nueva rama se ejecuta el comando>:

git branch [nombre]

Para ver todas las ramas existentes se ejecuta:

git branch --all

Para cambiar de posición de una rama a otra se ejecuta el comando:

git checkout [nombre de la rama]

La conexión de una rama a otra se llama merge para hacer merge nos posicionamos en la rama a la que queremos traer los cambios y ejecutamos el comando:

git merge [nombre de la rama de origen]

Para traer un commit de una rama a otra se usa el comando:

git cherry-pick [id del commit]

Github

De git a github

Para conectar con github primero configuramos la llave ssh estando en el home del directorio ejecutamos el código:

ssh-keygen -t rsa -b 4096 -C "correo"

Aceptar el directorio donde se guarda la contraseña

Crear una contraseña adicional ( passphrase )

Revisar que el servidor de llaves ssh esta prendido para esto ejecutamos el comando:

eval $(ssh-agent -s)

Después agregar la llave con el comando:

ssh-add ~/.ssh/id_rsa

Para copiar un repositorio hacia github primero creamos el repo en github y copiamos la url ssh después agregamos a git el origen ejecutando el comando:

git remote add origin [direccion ssh]

Primero hacemos el primer pull con el comando:

git pull origin master --allow-unrelated-histories

Después hacemos el push con el comando:

git push origin master

En github no deberían ir las imágenes estas deberían ir en un content delevery network ejemplo: imgur.com

Para que git ignore ciertos archivos que no queramos debemos crear un archivo .gitignore y dentro de este escribimos los formatos a ignorar o el nombre especifico de un archivo o carpeta.

En todos los proyectos debería existir un archivo README.md donde se detalle el proyecto.

Para hacer un fork a un repositorio agregamos un origen nuevo normalmente llamado upstream desde este nos traemos los cambios desde el repositorio original a nuestro local haciendo pull y después hacemos push a nuestro repositorio en github.
